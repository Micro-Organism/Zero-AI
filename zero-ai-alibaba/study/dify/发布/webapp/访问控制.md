# 访问控制

> 学习来源：https://docs.dify.ai/zh/use-dify/publish/webapp/web-app-access

## 一、概述

Web 应用访问控制决定了谁可以使用你发布的应用程序。默认情况下，新应用仅限特定团队成员访问——你可以精确选择谁有权限访问。

**信息提示**：只有工作空间所有者、管理员和编辑者角色才能创建和发布 Web 应用。

## 二、访问权限类型

从工作室 → Web 应用访问权限配置访问权限，或在编辑应用时从发布面板进行配置。

Dify Enterprise 提供四个访问级别：

### 2.1 平台内所有成员

你的 Dify Enterprise 工作空间中的任何成员都可以访问该应用。用户必须使用其工作空间凭据进行身份验证——密码、验证码或 SSO。

成员可以通过直接 URL 或工作空间 Explorer 页面访问应用。

**警告**：如果你从 Dify Enterprise v2.7.x 或更早版本升级且启用了 Web App SSO，你的应用在 v2.8.x 升级期间会自动切换为**已认证的外部用户**权限。

### 2.2 平台内特定成员

**新应用的默认设置。**限制访问权限给工作空间内选定的组或个人成员。非常适合部门特定工具或敏感数据应用。

**警告**：如果没有选择任何组或成员，任何人都无法访问你的应用——包括你自己。

#### 配置方式

**按组**：
- 添加整个组以实现自动权限管理
- 当有人加入组时，他们获得应用访问权限
- 当他们离开时，访问权限被撤销

**按个人**：
- 选择特定的个人成员
- 精确控制访问权限

**信息提示**：工作空间所有者、管理员和编辑者始终可以编辑工作空间中的任何应用。但是，他们仍需要被明确添加到访问列表中才能使用已发布的 Web 应用。

### 2.3 已认证的外部用户

你的 Dify Enterprise 工作空间外的用户可以通过 SSO 身份验证访问应用。管理员通过第三方身份提供商管理外部用户，将他们与内部工作空间数据分开。

#### 适用场景

- **大型企业**：IT 部门构建应用，其他部门无需加入 Dify 即可使用
- **外部合作伙伴**：为供应商、承包商或客户提供 AI 服务
- **客户支持**：面向公众的产品帮助和咨询工具

**信息提示**：如果此选项被禁用，请联系你的 Dify 管理员配置 Web 应用外部用户认证。

### 2.4 任何人

**无需身份验证。**任何拥有 URL 的人都可以立即访问你的应用。用于公开演示、客户工具或开放资源。

## 三、查找你的应用

团队成员可以在工作空间 Explorer 页面看到所有可访问的应用。

## 四、常见问题

### 4.1 权限更改是否需要重新发布？

不需要。访问权限更改会立即生效，无需重新发布应用。

### 4.2 如何查看谁有访问权限？

在应用的访问控制设置中，你可以查看所有被授予访问权限的用户和组。

### 4.3 我应该选择哪个权限级别？

- **内部工具**：使用"平台内特定成员"或"平台内所有成员"
- **外部服务**：使用"已认证的外部用户"或"任何人"
- **敏感数据**：使用"平台内特定成员"并仔细选择成员

### 4.4 Web 应用权限是否影响 API 访问？

不影响。Web 应用访问控制和 API 访问是独立的。API 访问通过 API Key 控制，不受 Web 应用访问权限影响。

## 五、关键要点

1. **精确控制**：可以精确控制谁可以访问你的应用
2. **灵活配置**：支持按组、按个人或公开访问
3. **即时生效**：权限更改立即生效，无需重新发布
4. **独立管理**：Web 应用权限和 API 访问权限是独立的
5. **安全第一**：默认情况下限制访问，需要明确授权
