# Web App 设置

> 学习来源：https://docs.dify.ai/zh/use-dify/publish/webapp/web-app-settings

## 一、概述

Web App 设置控制你发布的应用程序对最终用户的外观和行为。每个 Dify 应用程序都会自动生成一个适应不同设备和屏幕尺寸的 Web 界面。

## 二、设置工作原理

你的 Web 应用反映了你应用程序的当前配置。当你修改设置并点击"发布"时，更改会立即流向用户正在访问的实时 Web 应用程序。

**信息提示**：Web 应用默认情况下无需用户账户或登录即可工作，这使它们非常适合公共工具和演示。

## 三、品牌和外观

让你的 Web 应用看起来专业且易于识别：

### 3.1 应用身份
- 设置你的应用名称、描述和图标，以创造清晰的第一印象

### 3.2 视觉设计
- 选择与你的品牌匹配的颜色、主题和布局选项

### 3.3 语言设置
- 为你的目标受众配置界面语言

### 3.4 法律页面
- 添加版权信息和隐私政策链接以符合合规要求

## 四、基本品牌元素

### 4.1 应用图标和名称

- 你的图标会出现在浏览器标签页中，以及用户收藏你的应用时
- 选择一个清晰、易于识别的名称来解释你的应用功能
- 图标可以是图片或表情符号——选择符合你应用个性的

### 4.2 描述和消息传达

- 编写简洁的描述，帮助用户理解你应用的用途
- 此文本会出现在应用的着陆页和搜索结果中
- 为获得最佳效果，请保持在 160 个字符以内

### 4.3 视觉一致性

- 选择与你的品牌一致的颜色
- 在选择明亮或暗色主题时考虑你的受众
- 在不同设备上测试你的应用，确保在各处都看起来良好

## 五、访问和隐私控制

控制谁可以使用你的应用以及如何使用：

### 5.1 公共访问（默认）
- 默认情况下，Web 应用是公开的
- 任何拥有 URL 的人都可以访问

### 5.2 访问控制
- 可以限制访问权限给特定用户或组
- 需要身份验证才能访问

### 5.3 隐私设置
- 配置数据收集和使用策略
- 设置隐私政策链接

## 六、功能继承

你的 Web 应用会自动包含你在应用程序配置中启用的功能：

### 6.1 交互功能

- 对话开启器和建议的后续操作
- 对话前表单用于上下文收集
- 语音输入和语音转文字能力
- 来源引用与归属和参考链接
- 反馈收集和评分系统

### 6.2 功能性

- 所有工作流步骤和 AI 模型配置
- 知识库集成和工具连接
- 自定义提示词和响应格式
- 速率限制和使用控制

**警告**：在你的应用配置中禁用功能会立即从发布的 Web 应用中移除它们。

## 七、应用类型和行为

你的 Web 应用会根据你的应用程序类型自动适应其界面：

### 7.1 对话应用程序

- **界面**：具有消息历史记录的对话式风格
- **功能**：持久会话、对话管理、实时响应
- **最适合**：客户支持、咨询工具、交互式助手

### 7.2 工作流应用程序

- **界面**：基于表单的输入界面
- **功能**：单次执行、批处理、结果管理
- **最适合**：数据处理、内容生成、自动化任务

## 八、发布前检查清单

在分享你的 Web 应用之前：

1. **测试用户体验**：在不同设备和浏览器上尝试你的应用，确保运行顺畅
2. **配置基本设置**：设置你的应用名称、图标、描述和任何必需的法律信息
3. **审查访问控制**：决定你的应用应该是公开的还是需要身份验证
4. **监控使用情况**：检查应用使用情况和用户反馈，识别需要改进的领域

**提示**：从最小设置开始，根据用户反馈进行迭代。你可以随着了解人们如何使用你的应用而随时更新配置。

## 九、关键要点

1. **即时生效**：设置更改在发布后立即生效
2. **自动适应**：Web 应用自动适应不同设备和屏幕尺寸
3. **功能继承**：自动包含应用配置中的所有功能
4. **灵活配置**：可以自定义品牌、外观和访问控制
5. **无需额外开发**：无需编写代码即可创建专业的 Web 应用
