# 分享你的 AI 应用

> 学习来源：https://docs.dify.ai/zh/use-dify/publish/README

## 一、概述

你已经在 Dify 中构建了出色的应用。现在让我们将它交付给你的用户。每个 Dify 应用都会自动以多种方式提供服务——选择最适合你情况的方式。

## 二、从 Web 应用开始

通过 Web 应用分享是你最快的路径。这些应用在你创建任何应用时会自动生成，无需设置即可立即工作。

### 2.1 快速开始步骤

1. **点击发布**：在你的应用中点击"发布"以激活最新版本
2. **复制 URL**：在发布部分找到你的 Web 应用链接
3. **立即分享**：将链接发送给用户——他们可以立即开始使用你的应用

**提示**：Web 应用在任何设备上都能工作，并自动适应屏幕尺寸。无需应用商店审批或安装。

## 三、发布选项

Dify 提供四种主要的发布方式：

### 3.1 Web 应用
- **特点**：即时、可分享的应用
- **适用场景**：非常适合测试想法或直接为最终用户提供服务
- **详情**：参见 [WebApp 学习文档](./webapp/)

### 3.2 API 集成
- **特点**：将 AI 嵌入到你现有的产品中
- **适用场景**：完全控制用户体验和数据流
- **详情**：参见 [API集成学习文档](./Dify-API集成学习.md)

### 3.3 网站嵌入
- **特点**：向任何网站添加聊天组件
- **适用场景**：非常适合客户支持或潜在客户筛选
- **详情**：参见 [网站嵌入学习文档](./webapp/网站嵌入.md)

### 3.4 MCP 服务器
- **特点**：连接到 Claude Desktop 和 Cursor 等 AI 工具
- **适用场景**：非常适合开发工作流
- **详情**：参见 [MCP服务器学习文档](./MCP服务器.md)

## 四、发布工作原理

当你发布应用时，Dify 会在所有渠道激活你的最新配置。以下是幕后发生的情况：

- **Web 应用**：立即更新新功能和响应
- **API 端点**：提供最新的模型和工作流配置
- **嵌入式组件**：反映更新的样式和行为
- **MCP 服务器**：提供对当前应用功能的访问

**警告**：发布会用当前配置替换你的实时应用。用户将在下次交互中立即看到更改。

## 五、选择你的方法

根据你的需求选择合适的方式：

- **我想直接发布** → Web 应用
- **我正在构建一个产品** → API 集成
- **我有一个现有网站** → 网站嵌入
- **我想要扩展 AI 工具** → MCP 服务器

## 六、发布最佳实践

在分享你的应用之前，请确保你已配置这些设置：

### 6.1 应用描述
- 帮助用户了解你的应用功能

### 6.2 图标和品牌
- 使你的应用易于识别且专业

### 6.3 访问控制
- 决定你的应用应该是公开的还是需要身份验证

### 6.4 速率限制
- 保护你的应用免受过度使用（对于 API 访问尤其重要）

**检查清单**：所有发布方法都使用相同的应用配置。设置一次，随处发布。

## 七、关键要点

1. **统一配置**：所有发布方式共享相同的应用配置
2. **即时生效**：发布后更改立即生效
3. **多渠道支持**：一次配置，多种发布方式
4. **无需额外设置**：Web 应用自动生成，无需额外配置
