## 四、自动发现子页面的方法

### 推荐方案：通过侧边栏快照提取

#### 步骤
1. **访问任意页面**：访问目标模块下的任意页面（如 Graph Core 的 quick-start）
2. **获取页面快照**：使用 `browser_snapshot` 获取页面结构
3. **提取侧边栏导航**：从快照的 YAML 结构中提取侧边栏导航部分
4. **识别链接元素**：查找所有 `link` 元素及其 `/url` 属性
5. **处理折叠目录**：
   - 对于折叠的目录（如"核心功能"、"示例"），先点击展开按钮
   - 再次获取快照，提取新展开的链接
6. **合并和去重**：合并所有链接，去除重复项

#### 优点
- ✅ **准确**：直接从页面 DOM 提取，确保链接正确
- ✅ **完整**：可以获取所有子页面链接，包括折叠目录中的
- ✅ **自动化**：可以通过脚本实现，无需手动操作

#### 缺点
- ⚠️ **需要处理折叠状态**：需要识别并展开折叠的目录
- ⚠️ **需要解析 YAML**：需要解析快照的 YAML 结构

#### 实现示例
```javascript
// 伪代码示例
1. navigate("https://java2ai.com/docs/frameworks/graph-core/quick-start")
2. snapshot = browser_snapshot()
3. 从 snapshot 中提取侧边栏导航
4. 识别所有 button 元素（折叠目录）
5. 对于每个折叠的 button：
   - 点击展开
   - 再次获取快照
   - 提取新展开的链接
6. 合并所有链接，生成文件列表
```
